Équilibrage de Charge SD-WAN vs Load Balancing Classique

SD-WAN WAN Load Balancing : Ici, on répartit le trafic réseau sortant d'une entreprise sur plusieurs liaisons WAN hétérogènes (ex : MPLS, Fibre, 4G, Internet). L'objectif est d'optimiser l'utilisation des liens physiques : plus de bande passante, moins de congestion, tolérance aux pannes, économies.
— C'est le WAN qu'on équilibre, pas des serveurs ou services.

  • Chaque paquet peut partir sur un lien différent, selon un algorithme (RR, pondéré, etc.)
  • Visuellement, on voit chaque lien WAN utilisé dans la proportion voulue
  • Algorithmes : Round Robin simple (tour à tour), Round Robin pondéré (liens plus ou moins utilisés selon leur poids/capacité)

Load Balancing Classique : On répartit les requêtes (ex : HTTP) entre plusieurs serveurs pour augmenter la capacité d'un service web.
Ici, ce n'est pas le sujet ! On équilibre le réseau physique, pas les serveurs.
